Migração de WordPress para Site Estático com GitHub Pages

Resumo:
Este projeto documenta o processo completo de migração de um site WordPress hospedado em servidor próprio para uma arquitetura de site estático utilizando GitHub Pages. A solução elimina dependências de backend, reduz custos de hospedagem e mantém controle total via Git.

🧠 Contexto

O site original (WordPress) foi convertido para HTML estático, preservando estrutura, conteúdo e identidade visual. A nova arquitetura utiliza apenas HTML, CSS e JavaScript puro, com dados dinâmicos gerenciados via JSON e GitHub API.

⚙️ Principais Etapas

1. Extração e Conversão

  • Exportação de posts, páginas e mídias
  • Conversão para HTML estático
  • Manutenção da estrutura original de diretórios

2. Setup no GitHub Pages

  • Criação de repositório
  • Publicação via branch master
  • Fluxo Git: add → commit → pull → push

3. Ajustes Pós-Migração

  • Correção de paths de imagens (thumbs)
  • Implementação de widget de clima dinâmico
  • Correção de erros JavaScript (DOM vs innerHTML)
  • Cache busting com query string (?v=N)
  • Ajuste de base path em scripts
  • Correção de layout responsivo
  • Sincronização de carregamento de posts

4. Painel Administrativo (SPA)

  • Interface web para criação de posts
  • Upload e organização automática de imagens
  • Integração direta com GitHub API
  • Atualização automática de posts.json

🏗️ Arquitetura Final

  • index.html → homepage
  • posts.json → fonte de dados
  • sidebar.js → renderização dinâmica
  • /content/uploads/ → mídia organizada por data
  • /YYYY/MM/DD/slug/ → posts
  • /admin/ → painel de controle

🚀 Tecnologias Utilizadas

  • GitHub Pages (hospedagem)
  • JavaScript Vanilla
  • GitHub REST API
  • Open-Meteo + Nominatim (clima/geolocalização)
  • CSS customizado

✅ Benefícios da Migração

  • Zero custo de hospedagem
  • Maior performance (sem backend)
  • Controle total via Git
  • Segurança (sem PHP/MySQL)
  • Simplicidade operacional
